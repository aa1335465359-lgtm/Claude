# Claude Local Web Client

这是一个模仿 Claude 官方界面的本地化网页客户端，旨在提供流畅、隐私、功能丰富的 AI 对话体验。

## 🌟 核心特性

- **极致的 UI 体验**：
  - 1:1 复刻 Claude 官方深色模式界面，包含动效、字体和布局细节。
  - 支持 **Markdown 渲染**、代码高亮、表格显示。
  - 提供了类似官方的“思考过程”折叠面板，支持查看 AI 的思维链。

- **强大的模型支持**：
  - 内置多种模型配置（如 Claude 3.7 Sonnet, Opus 4.5/4.6 等）。
  - **双模式思考**：
    - ⚡ **快速模式 (Quick Mode)**：默认的自适应模式，响应迅速，适合日常问答。
    - 🧠 **深度思考 (Deep Thinking)**：强制开启长思维链，分配高达 32k 的思考 Token（总输出支持 64k），适合解决复杂的编程、数学或逻辑推理问题。

- **本地化与隐私**：
  - **纯前端运行**：所有逻辑在浏览器中执行，不依赖后端中转服务（直接连接配置的 API）。
  - **本地存储**：对话记录存储在浏览器的 `localStorage` 中，不会上传到任何第三方数据库（除了发送给 AI 接口本身）。这意味着清除浏览器缓存会清空聊天记录。

## 💡 上下文是如何处理的？

你可能好奇 AI 是如何“记住”我们聊过的内容的。在这个客户端中，上下文处理遵循以下逻辑：

1.  **无状态接口**：AI 模型本身是不记事的（Stateless）。
2.  **全量携带**：当你点击发送时，前端代码会从本地存储中读取当前会话的**完整历史记录**（包括你之前的提问、AI 的回答、上传的图片和文件）。
3.  **打包发送**：这些历史记录被打包成一个标准的 Message List 数组，一次性发送给 API。
4.  **Token 优化**：
    - 对于超长对话，主要受限于模型的上下文窗口（Context Window）。
    - 包含图片时，图片会被转换为 Base64 编码随请求发送。
    - 包含文本文件时，内容会被读取并包裹在 XML 标签中发送给模型。

简而言之：**“记忆”其实是你每次重新发给他的“剧本”。**

## 📂 文件与多模态

- **附件支持**：支持拖拽或点击上传图片（支持多图）和文本文件（.txt, .md, .js, .json 等）。
- **代码工件**：自动识别 Markdown 代码块，提供一键复制功能和优雅的语法高亮。

## ⚙️ 隐藏功能

- **设置面板**：为了保持界面简洁，调试设置入口默认隐藏。你需要**连续点击 10 次**右上角的设置图标（⚙️）才能打开面板，查看当前的 API 连接状态、Base URL 和 Token 配置。
